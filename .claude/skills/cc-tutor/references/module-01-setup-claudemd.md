{team}의 마케터들은 리드 세그먼트 분석, 콘텐츠 소재 리서치, 캠페인 기획, CRM 데이터 분석을 동시에 처리합니다. Claude Code는 이 모든 업무를 함께할 **새로운 팀원**입니다.

오늘은 이 팀원에게 **첫 브리핑**을 하는 겁니다. 설치하고, 인사하고, 첫 업무를 맡겨봅니다.

**Before:** 매번 "존댓말로 써줘", "표로 정리해줘", "타겟 기업 Pain Point부터 시작해" 반복

## EXPLAIN

### Part A: Claude Code 기본

#### Claude Code란?

터미널에서 `claude` 한 글자를 치면 시작되는 AI 어시스턴트입니다.

| 항목 | 내용 |
|------|------|
| 비유 | **새로 합류한 주니어 마케터** — 지시하면 리서치, 문서 작성, 파일 정리를 해줍니다 |
| 시작법 | 터미널에서 `claude` 입력 |
| 특징 | 파일을 직접 읽고 쓸 수 있음, 명령어를 실행할 수 있음, 대화하면서 업무를 처리 |

#### 터미널이란?

> 📖 glossary 참조: CLI = "광고 대시보드에 직접 명령"

마우스로 클릭하는 대신, 텍스트로 컴퓨터에 직접 지시하는 창입니다.
GA4에서 쿼리를 직접 날리듯, 명령어로 바로 실행합니다.

**터미널 여는 법:**
- **Windows:** `Win + R` → `cmd` 입력 → Enter (또는 시작 메뉴에서 "PowerShell" 검색)
- **Mac:** `Cmd + Space` → "터미널" 검색 → Enter

#### 권한 시스템

Claude Code는 파일 읽기/쓰기, 명령어 실행 전에 허락을 구합니다.
새 팀원이 {name} {title}님에게 "이거 해도 돼요?" 확인받는 것과 같습니다.

| 권한 | 설명 |
|------|------|
| Allow once | 이번 한 번만 허락 |
| Allow always | 이 작업은 항상 허락 (자주 하는 작업에 편리) |
| Deny | 거절 |

> 💡 처음에는 **Allow once**로 시작하세요. 뭘 하는지 눈으로 확인하는 습관이 중요합니다.

#### 에디터 (선택사항)

Claude Code는 터미널에서 `claude`만 치면 됩니다. 파일을 눈으로 보면서 작업하고 싶다면:

| 에디터 | 추천 대상 | 사이트 |
|--------|----------|--------|
| Antigravity | 처음 시작하는 분 | antigravity.google |
| Cursor | 코드 편집도 할 분 | cursor.com |
| VSCode | 이미 쓰고 계신 분 | code.visualstudio.com |

> "이 과정에서는 에디터 없이 터미널만으로 충분합니다."

#### 꿀팁: 숨겨진 폴더 보기

`.`으로 시작하는 폴더(`.claude` 등)는 기본적으로 안 보입니다.

**Windows:** 파일 탐색기 → 상단 "보기" → "숨김 항목" 체크
**Mac:** Claude Code에게 `Finder에서 숨겨진 폴더도 볼 수 있게 설정해줘` 라고 요청

설정 후 폴더 열기:
- **Windows:** 탐색기에서 해당 폴더로 이동
- **Mac:** `open ./`

> `.claude/skills/` 폴더 안에 지금 배우고 있는 교안이 들어있습니다!

#### Output Style 설정

로그인 후 `/output-style`을 입력하고 **Explanatory**를 선택하세요.
Claude가 왜 그렇게 하는지 설명을 함께 해줍니다. 배우는 단계에서 가장 적합한 모드입니다.

### Part B: CLAUDE.md + Auto Memory

#### 1. CLAUDE.md — 내가 쓰는 지시문

| 항목 | 내용 |
|------|------|
| 비유 | **팀 캠페인 가이드라인** — Claude가 매 대화 시작할 때 읽는 규칙서 |
| 근본 원리 | 시스템 프롬프트 — AI가 대화를 시작할 때 가장 먼저 읽는 지시문 |
| 예시 | "항상 존댓말로", "제안서는 타겟 기업 Pain Point부터 시작", "금액은 예상 전환율 포함" |

```
세션 시작
  │
  ▼
┌─────────────┐     ┌─────────────┐
│ CLAUDE.md   │────▶│ 시스템       │────▶ Claude가 규칙을 아는 상태로 대화 시작
│ (내 지시문)  │     │ 프롬프트     │
└─────────────┘     └─────────────┘
  항상 읽힘            매번 자동 주입
```

#### 2. Auto Memory — Claude가 스스로 적는 메모

| 항목 | 내용 |
|------|------|
| 비유 | **Claude의 업무 수첩** — 일하면서 적어두는 자기 노트 |
| CLAUDE.md와 차이 | CLAUDE.md는 **내가** Claude에게 주는 규칙. Auto Memory는 **Claude가** 스스로 적는 메모 |
| 저장 위치 | `~/.claude/projects/<프로젝트>/memory/` |

**Auto Memory가 기록하는 것들:**
- 프로젝트 패턴: 자주 쓰는 파일 구조, 명명 규칙
- 디버깅 인사이트: 까다로운 문제의 해결법
- 내 선호사항: 소통 방식, 워크플로우 습관

#### 3. 계층 구조 — 용도에 따라 나눠서 쓰기

CLAUDE.md는 여러 레벨에 둘 수 있고, **각 레벨마다 넣을 내용이 다릅니다:**

| 레벨 | 위치 | 넣을 내용 | 비유 |
|------|------|----------|------|
| **개인 글로벌** | `~/.claude/CLAUDE.md` | 나에 대한 정보, 소통 선호 | **내 명함 + 업무 스타일** |
| **프로젝트** | 프로젝트 폴더의 `CLAUDE.md` | 이 프로젝트의 업무 규칙 | **프로젝트별 가이드라인** |

> 💡 **개인 글로벌**은 어떤 프로젝트에서든 항상 적용됩니다. **프로젝트**는 해당 폴더에서만 적용됩니다.

### Part B: CLAUDE.md 작성

**실습 1: 개인 글로벌 CLAUDE.md 만들기 (나에 대한 정보)**

먼저 **나는 누구인지** AI에게 알려줍니다. 이 파일은 어떤 프로젝트에서든 항상 적용됩니다.

Claude Code에 이렇게 요청하세요:

    ~/.claude/CLAUDE.md 파일에 다음 내용을 넣어줘:

    # 나에 대한 정보
    - 이름: {name}
    - 소속: {division} {team}
    - 역할: B2B 마케터

    # 소통 선호
    - 항상 한국어로 대화
    - 설명할 때 비유와 예시를 함께 사용
    - 비개발자 눈높이에 맞춰 설명

> 이제 Claude는 **어떤 프로젝트에서든** {name} {title}님이 누구인지 알고 대화를 시작합니다.

**실습 2: 프로젝트 CLAUDE.md 만들기 (업무 규칙)**

이번에는 **이 프로젝트의 규칙**을 설정합니다. `/init` 명령어로 시작하면 편합니다:

    /init

> ⚠️ **주의:** 이미 CLAUDE.md가 있는 프로젝트에서 `/init`을 실행하면 기존 내용이 덮어씌워질 수 있습니다. 기존 파일이 있다면 `/init` 대신 직접 수정하세요.

자동 생성된 CLAUDE.md에 마케팅 규칙을 추가합니다:

    CLAUDE.md에 다음 규칙들을 추가해줘:

    ## 캠페인 기획 규칙
    - 항상 존댓말로 작성
    - 캠페인 기획서 구조: 타겟 기업 Pain Point → 캠페인 KPI → 캠페인 플로우 → 콘텐츠 구성 → 캠페인 일정·예산 → 예상 ROI
    - 금액은 예상 전환율 포함
    - 고객사명은 정식 명칭 사용 (예: 정식 기업명)

    ## 콘텐츠 유형 분류
    - AI/DT: AI 전문가, DT 마케터
    - 리더십: 리더십 코치, 조직 개발 전문가
    - 직무: 직무별 현업 실무자

> 이 규칙은 **이 프로젝트 폴더에서만** 적용됩니다. 다른 프로젝트에서는 다른 규칙을 쓸 수 있어요.

**실습 3: /memory로 확인**

    /memory

개인 글로벌 CLAUDE.md와 프로젝트 CLAUDE.md가 각각 잘 설정되었는지 확인해보세요.

**실습 4: Auto Memory 직접 체험**

Claude Code에 이렇게 말해보세요:

    나는 캠페인 기획할 때 항상 고객의 비즈니스 임팩트를 먼저 강조하는 걸 좋아해. 이걸 기억해둬.

`/memory`로 저장된 내용을 확인해보세요. 실습 1에서 넣은 **개인 정보**와 달리, 이건 **Claude가 스스로 메모**한 것입니다.

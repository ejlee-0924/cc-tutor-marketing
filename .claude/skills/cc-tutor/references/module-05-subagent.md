# Module 5: Subagent

> 공식 문서: https://code.claude.com/docs/ko/sub-agents

## WHY

{name} {title}님이 {colleagues}에게 각각 다른 리서치를 동시에 요청하는 상황을 떠올려보세요.

- {colleagues[0]}에게: "삼성전자 최근 교육 트렌드 조사해줘"
- {colleagues[1]}에게: "AI 교육 콘텐츠 주제 3개 리서치해줘"

이 두 작업은 서로 독립적이라 동시에 진행할 수 있죠. **Subagent가 바로 이겁니다** — AI 팀원 여러 명이 동시에 각자 맡은 일을 합니다.

**Before:** 리서치 3개를 하나씩 순서대로 요청 → 30분
**After:** 3개를 동시에 돌림 → 10분

## EXPLAIN

### Subagent란?

> 📖 glossary 참조: Subagent = "팀원에게 동시 리서치 지시"

메인 Claude가 "보스" 역할을 하고, 여러 Subagent(팀원)에게 각각 독립적인 작업을 분배합니다.

```
{name} {title}님 (지시)
    │
    ▼
메인 Claude (보스)
    │
    ├──▶ Subagent 1: 시장조사 ──▶ 결과 보고
    ├──▶ Subagent 2: 콘텐츠 소재 리서치 ──▶ 결과 보고
    └──▶ Subagent 3: 경쟁사 분석 ──▶ 결과 보고
    │
    ▼
통합 결과 정리
```

### 언제 쓰나?

| 상황 | Subagent 사용 | 이유 |
|------|-------------|------|
| 마케팅 콘텐츠 3건을 각각 조사 | ✅ | 독립적인 작업 3개 |
| 캠페인 기획서의 각 섹션 작성 | ✅ | 섹션별 독립 작업 |
| 이전 작업 결과를 바탕으로 다음 작업 | ❌ | 순차적 의존성 |

**핵심:** 각 작업이 **서로 독립적**이면 Subagent, **앞의 결과가 필요**하면 순차 실행.

### Subagent의 특징

- 각 Subagent는 **독립적인 맥락**에서 작업 (서로 간섭 없음)
- 메인 Claude에게 결과를 **보고** (보스에게 보고하듯)
- 여러 작업이 **동시에** 진행되어 시간 절약

## EXECUTE

아래 순서대로 실행해보세요:

**실습: 마케팅 콘텐츠 3건 병렬 리서치**

Claude Code에 이렇게 요청해보세요:

    아래 3개 마케팅 콘텐츠 주제를 동시에 조사해서 비교표를 만들어줘.
    각각 독립적으로 리서치해서 병렬로 진행해줘.

    1. "AI 업무 자동화 성공 사례집" — 대기업 타겟
    2. "2026 AI 교육 트렌드 리포트" — 중견기업 타겟
    3. "AI 도입 ROI 가이드" — 스타트업 타겟

    각각에 대해:
    - 타겟 기업이 관심 가질 포인트
    - 콘텐츠 핵심 구성 (목차 5개)
    - 예상 다운로드 후 상담 전환율
    - 배포 채널 추천

    최종적으로 3개를 비교하는 표로 정리해줘.

Claude가 Subagent를 사용하여 3개 리서치를 동시에 진행하는 것을 관찰해보세요. Agent 도구를 호출하는 것이 보일 겁니다.

**추가 실습 (선택): 제안서 섹션 병렬 작성**

    SK하이닉스 반도체 엔지니어 대상 AI 활용 마케팅 캠페인 기획서를 작성해줘.
    아래 3개 섹션을 동시에 병렬로 작성해줘:
    1. 시장 분석 및 교육 필요성
    2. 캠페인 상세 구성
    3. 예상 ROI 및 ROI

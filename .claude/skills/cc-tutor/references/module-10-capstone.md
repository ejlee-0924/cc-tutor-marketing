# Module 10: 종합 실전 프로젝트

> M1~M9에서 배운 기능을 조합하여 본인의 실제 반복 업무 1건을 E2E 자동화합니다.

## WHY

{name} {title}님, 지금까지 9개 모듈을 통해 Claude Code의 핵심 기능을 익혔습니다.

하지만 진짜 실력은 **"배운 기능을 내 업무에 조합할 수 있느냐"**에서 갈립니다. 이 모듈에서는 본인이 가장 시간을 많이 쓰는 반복 업무 1건을 선택하여, M1~M9의 기능을 자유롭게 조합한 E2E 자동화 프로젝트를 완성합니다.

**Before:** 기능은 각각 배웠지만, 실제 업무에 어떻게 조합하는지 감이 안 옴
**After:** 본인 업무 1건이 완전히 자동화되고, 이 경험을 다른 업무에도 확장 가능

## EXPLAIN

### 프로젝트 선정 기준

좋은 프로젝트를 고르는 3가지 기준:

| 기준 | 설명 | 예시 |
|------|------|------|
| **반복성** | 주 1회 이상 반복하는 업무 | 주간 캠페인 리포트, 월간 성과 분석 |
| **다단계** | 2개 이상 단계가 있는 흐름 | 데이터 수집 → 분석 → 리포트 → 공유 |
| **시간 소요** | 현재 30분 이상 걸리는 업무 | 리드 세그먼트 분석 → 캠페인 기획 |

### 추천 프로젝트 예시

**예시 A: 콘텐츠 제작 파이프라인 자동화**
- 트렌드 수집(MCP) → 초안 작성(CLAUDE.md 규칙) → 병렬 검수(Subagent) → 채널별 배포(MCP/Slack)
- 활용 모듈: M1, M2, M3, M4, M8

**예시 B: 캠페인 성과 분석 자동화**
- 데이터 수집(MCP) → 분석(파일 도구) → 리포트 생성(스킬) → Slack 공유(MCP)
- 활용 모듈: M2, M3, M5, M6, M8

**예시 C: 본인 업무 자유 선택** ← 권장
- 본인이 가장 시간을 많이 쓰는 반복 업무 1건을 직접 선택
- 전 모듈 자유 조합

### 프로젝트 진행 순서

1. 업무 선정 → 2. 단계 분해 → 3. 모듈 매핑 → 4. 구현 → 5. 테스트 → 6. 문서화

## EXECUTE

---

### 1단계: 프로젝트 선정

📋 아래를 복사해서 입력하세요:

    내가 매주 반복하는 마케팅 업무 중에서 자동화하면 가장 효과가 큰 것을 골라보고 싶어.
    후보를 3개 제시해줘. 각각 예상 소요 시간, 자동화 가능성, 활용할 Claude Code 기능을 정리해줘.

---

### 2단계: 단계 분해

선정한 업무를 Claude Code와 함께 단계별로 쪼개세요:

📋 아래를 복사해서 입력하세요:

    이 업무를 자동화하려면 어떤 단계로 나눌 수 있을까?
    각 단계에서 어떤 Claude Code 기능(파일 도구, MCP, Subagent, Skill 등)을 쓸 수 있는지 매핑해줘.

---

### 3단계: 구현

Plan 모드로 시작하는 것을 추천합니다:

📋 아래를 복사해서 입력하세요:

    Plan 모드로 전환해서, 이 자동화 프로젝트의 실행 계획을 짜줘.

계획이 확정되면 단계별로 구현합니다. 필요하다면 스킬로 만들거나(M7), 에이전트 팀을 구성(M8)하세요.

---

### 4단계: 테스트 + 문서화

📋 아래를 복사해서 입력하세요:

    이 자동화 워크플로우를 테스트하고, 사용법을 README로 정리해줘.
    팀원이 바로 따라할 수 있도록 단계별 안내를 포함해줘.

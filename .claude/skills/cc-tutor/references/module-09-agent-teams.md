# Module 9: Agent Teams

> 공식 문서: https://code.claude.com/docs/ko/agent-teams

## WHY

{name} {title}님, 큰 제안서 프로젝트는 혼자 하지 않죠. 리서처가 시장조사를 하고, 기획자가 캠페인 플로우을 짜고, QA가 검수합니다. **Agent Teams는 이런 프로젝트 TF를 AI로 구성하는 것**입니다.

**Before:** 캠페인 기획 → 검수 → 수정 → 재검수를 혼자 반복 (3일)
**After:** AI TF가 리서치→초안→검수까지 파이프라인으로 실행 (3시간)

## EXPLAIN

### Agent Teams란?

> 📖 glossary 참조: Agent Teams = "프로젝트 TF"

여러 AI 에이전트에게 각각 다른 역할을 부여하고, 하나의 프로젝트를 협업으로 완성하게 하는 것입니다.

```
{name} {title}님 (프로젝트 오너)
    │
    ▼
AGENTS.md (각 에이전트의 JD)
    │
    ├──▶ 시장조사 에이전트 ──▶ 산업 동향 + 경쟁사 분석
    ├──▶ 캠페인 기획 에이전트 ──▶ 리서치 기반 캠페인 기획서 초안
    └──▶ QA 에이전트 ──▶ 논리 흐름 + 누락 항목 + 톤 검수
    │
    ▼
검증된 제안서 완성
```

### AGENTS.md — 에이전트의 JD

| 항목 | 설명 |
|------|------|
| 비유 | 각 TF 멤버의 직무기술서(JD) |
| 역할 | 에이전트 이름, 전문 분야, 접근 권한, 업무 범위 |
| 위치 | 프로젝트 폴더의 `.claude/` 안에 작성 |

```markdown
# AGENTS.md 예시

## researcher
- 역할: 시장조사 및 경쟁사 분석
- 전문: 산업 동향, 교육 트렌드, 콘텐츠 소재 리서치
- 접근: 웹 검색, 문서 읽기

## writer
- 역할: 마케팅 캠페인 기획서 작성
- 전문: 캠페인 기획서 구조화, 톤앤매너, 캠페인 플로우 설계
- 접근: 리서치 결과, 템플릿, CLAUDE.md 규칙

## reviewer
- 역할: 제안서 품질 검수
- 전문: 논리 검증, 누락 체크, 고객 관점 검토
- 접근: 최종 제안서, 고객 요구사항 원본
```

### 에이전트 간 협업 방식

| 방식 | 설명 | 적합한 상황 |
|------|------|-----------|
| 순차형 | A→B→C 순서대로 | 캠페인 파이프라인 (리서치→작성→검수) |
| 병렬형 | A, B, C 동시에 | 독립적 리서치 3건 동시 진행 |
| 대립형 | A vs B → C가 종합 | 의사결정 검증 (찬성 vs 반대 → 판정) |

## EXECUTE

### 실습 9-1: 캠페인 파이프라인 TF 구성 [목표 A]

아래를 Claude Code에 요청하세요:

    SK하이닉스 타겟 반도체 AI 캠페인을 기획하려고 해.

    Agent Teams로 3명의 TF를 구성해서 캠페인 기획서를 만들어줘:

    1. 시장조사 에이전트
       - 반도체 산업의 AI 활용 동향
       - 경쟁사 교육 프로그램 분석
       - 최신 AI 교육 트렌드

    2. 캠페인 기획 에이전트
       - 리서치 결과를 바탕으로 캠페인 기획서 초안
       - 우리 팀 CLAUDE.md 규칙 따르기
       - 6섹션 구조

    3. QA 에이전트
       - 논리 흐름 검증
       - 누락 항목 체크
       - 톤앤매너 일관성 확인

    먼저 AGENTS.md를 작성하고, 실제로 End-to-End 실행해줘.

Agent Teams가 순차적으로 작업하는 과정을 관찰하세요. 리서치 → 초안 → 검수 흐름이 자동으로 진행됩니다.

### 실습 9-2: 의사결정 검증 [목표 B]

9-1에서 만든 캠페인 기획서를 3관점으로 검증합니다:

    방금 만든 SK하이닉스 캠페인 기획서를 3개 관점으로 검증해줘:

    1. 찬성 에이전트 — 이 캠페인 기획서의 강점과 설득 포인트를 정리
    2. 반대 에이전트 — 약점, 리스크, 고객이 반론할 수 있는 포인트를 지적
    3. 고객 관점 에이전트 — SK하이닉스 교육 담당자 입장에서 평가

    3개 관점을 종합해서, 캠페인 기획서를 최종 보강해줘.

{group_leader} 그룹장님 앞에서 발표하기 전에, AI로 미리 "품평회"를 여는 겁니다.

### 실습 9-3: 확장 설계 [리더 실습]

    이 캠페인 파이프라인 + 의사결정 검증 구조를,
    분기별 캠페인 10건을 동시에 처리할 수 있는 구조로 확장 설계해줘.

    각 제안서마다 3-에이전트 TF가 독립적으로 작동하고,
    전체를 관리하는 매니저 에이전트가 있는 구조로.

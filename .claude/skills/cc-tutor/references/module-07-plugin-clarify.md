# Module 7: Plugin + Clarify

> 공식 문서: https://code.claude.com/docs/ko/plugins

## WHY

{name} {title}님, 고객이 "좋은 교육 해주세요"라고 합니다. 이게 정확히 뭘 원하는 걸까요?

대상이 누구인지, 어떤 수준인지, 교육 후 기대하는 변화가 뭔지... 이런 모호한 요구사항을 구조화해주는 것이 **Clarify 플러그인**입니다.

**Before:** 고객 미팅에서 두루뭉실한 요청을 받고, 제안서 쓰다가 "이게 맞나?" 고민
**After:** Clarify로 빠짐없이 질문 → 구조화된 스펙으로 제안서 착수

## EXPLAIN

### Plugin이란?

> 📖 glossary 참조: Plugin = "외부 전문 컨설팅 펌"

Plugin은 Skill보다 더 독립적이고 강력한 외부 도구입니다.

| 항목 | Skill | Plugin |
|------|-------|--------|
| 비유 | 내부 강사 (가볍고 빠름) | 외부 전문 컨설팅 펌 (무겁지만 독립적) |
| 구조 | SKILL.md 파일 하나 | 독립적인 패키지 |
| 설치법 | `npx skills add` | `npx plugins add` |
| 특징 | 가볍고 빠름 | 복잡한 워크플로우 가능 |

### Clarify 플러그인

모호한 요구사항을 **구조화된 스펙**으로 변환해주는 플러그인입니다.

```
고객: "디지털 전환 교육 해주세요"
    │
    ▼
Clarify가 질문을 던짐:
    ├── "대상은 누구인가요?" → 전 직원 500명
    ├── "현재 디지털 역량 수준은?" → 초급
    ├── "교육 후 기대하는 변화는?" → AI 도구 활용 능력
    ├── "기간과 예산은?" → 2일, 3000만원
    └── "성과 측정 기준은?" → 업무 적용 사례 제출
    │
    ▼
구조화된 교육 스펙 완성
```

### Clarify의 3가지 모드

| 모드 | 설명 | B2B 활용 |
|------|------|---------|
| vague (모호함 정리) | 두루뭉실한 요청을 구체화 | "좋은 교육 해주세요" → 구체적 스펙 |
| unknown (미지 발견) | 모르는 것을 찾아냄 | 캠페인 기획서의 빈 곳, 빠진 관점 발견 |
| metamedium (관점 전환) | 다른 접근법 탐색 | "강의 말고 워크샵은?" "오프라인 말고 온라인은?" |

### skill-creator 플러그인 소개

> 모듈 8에서 본격적으로 다룹니다.

{name} {title}님이 10년간 쌓은 노하우를 AI가 따라하게 만드는 도구입니다. 여기서는 설치만 해두세요.

## EXECUTE

아래 순서대로 실행해보세요:

**실습 1: 플러그인 목록 확인**

    현재 설치된 플러그인 목록을 보여줘.

**실습 2: Clarify 체험 — 모호한 고객 요청 구조화**

Claude Code에 이렇게 요청하세요:

    고객이 이렇게 요청했어:
    "우리 회사 직원들 AI 교육 좀 해주세요. 요즘 다들 AI 쓰잖아요."

    이 요구사항을 Clarify로 구조화해줘.
    마케팅 캠페인 기획서를 쓸 수 있을 정도로 구체적으로 만들어줘.

Clarify가 질문을 던지면 하나씩 답해보세요. 최종적으로 구조화된 교육 스펙이 나옵니다.

**실습 3: Clarify로 고객 미팅 질문 리스트 뽑기**

    다음 주에 현대자동차 HR 담당자와 첫 미팅이야.
    "AI 활용 교육" 주제로 논의할 건데,
    미팅에서 꼭 확인해야 할 질문 리스트를 Clarify로 뽑아줘.

**실습 4: skill-creator 플러그인 설치 (모듈 8 준비)**

    skill-creator 플러그인을 설치해줘.

## LEADER-TIP

> 💡 **{name} {title}님을 위한 리더 팁**
>
> - **고객 미팅 전에 Clarify로 질문 리스트**를 뽑아가면, 한 번 미팅에서 필요한 정보를 다 얻을 수 있습니다
> - {team} 팀원들에게도 "고객 요청 받으면 먼저 Clarify 돌려봐"라고 가이드하세요
> - Clarify의 3가지 모드를 상황에 맞게 쓰세요:
>   - 첫 미팅 전: **vague** (요구사항 구체화)
>   - 캠페인 기획 중: **unknown** (빠진 관점 발견)
>   - 차별화 고민 시: **metamedium** (다른 교육 형식 탐색)
> - 📋 **요구사항 전문가 배지를 획득했습니다!** 이제 고객의 모호한 요청을 구조화할 수 있습니다
